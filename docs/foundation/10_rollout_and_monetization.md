# 10. Rollout, MVP, Monetization & Operations

## Rollout Phases

**Phase 1: Foundation** — finalize house style, define syllabus backbone, build database schema, build prompt stack, choose first subject

**Phase 2: Prototype** — automate one subject, produce 5–10 topics, generate drafts/assets, publish manually/semi-manually, refine prompts and QA

**Phase 3: Semi-automated publishing** — enable scheduled uploads, maintain human review gate, collect analytics, improve stability

**Phase 4: Expanded automation** — add all platforms, automate backlog management, improve feedback loop, reduce manual work to exception handling

**Phase 5: Full-scale operation** — continuous generation, performance-based topic selection, multilingual/multi-voice expansion, premium products

---

## MVP Recommendation

First version should include:
- **one subject only:** Statistics or Machine Learning
- one topic queue, one dossier format, one episode template
- one short-form video template, one long-form script template
- uploads to X and YouTube first
- TikTok after the pipeline is stable

---

## Monetization Layers
- platform creator revenue
- subscriptions, premium notes, paid newsletters
- courses, ebooks, curriculum bundles
- school/teacher resources
- sponsored educational partnerships later

Optimize for: originality, consistency, trust, reference quality, evergreen utility.

---

## Analytics Loop
Collect: impressions, views, watch time, completion rate, retention curve, reposts/shares, comments, saves, click-through, follower/subscriber growth.

Claude generates: hook performance summaries, analogy performance summaries, topic cluster performance, next topic recommendations.

---

## Definition of Done
A topic is complete only when:
- research dossier exists
- references are stored
- episode arc is approved
- scripts for all target platforms generated
- citation checks pass
- media assets rendered
- compliance checks pass
- content uploaded or scheduled
- analytics logging enabled
